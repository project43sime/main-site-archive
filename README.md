# main-site-archive

## 项目简介

**main-site-archive** 是一个用于归档和发布多个独立 HTML 页面的仓库。  
它不针对任何特定域名、网站或主题，仅作为 HTML 静态内容的存储与分发空间。  
每个页面由贡献者或维护者独立提交，内容涵盖广泛类别，适合浏览、参考或二次使用。

## 目录说明

仓库采用扁平化结构，所有 HTML 文件直接存放于根目录或按需分入子文件夹。  
典型目录布局如下：

```
main-site-archive/
├── index.html          # 仓库入口页面（可选）
├── page-1.html         # 独立页面示例
├── page-2.html
├── subfolder/
│   ├── some-page.html
│   └── another-page.html
└── README.md           # 本文件
```

- 文件名应简洁明了，反映页面内容（如 `about-us.html`、`contact-form.html`）。
- 子文件夹可用于按主题或日期组织页面。
- 不强制使用特定命名规则，但建议保持一致性。

## 页面归档说明

- 所有 HTML 页面均为独立文件，可脱离仓库直接通过浏览器打开。
- 页面样式、脚本和资源应尽量内联或使用相对路径引用，以确保便携性。
- 归档页面不包含任何恶意代码、跟踪器或外部链接（除非明确标注）。
- 页面内容可能随时间更新或移除，历史版本可参考 Git 提交记录。

## 维护说明

- 本仓库由社区或指定维护者管理，接受合理的页面提交请求。
- 提交新页面时，请确保：
  - 文件为有效的 HTML5 格式。
  - 不侵犯他人版权或隐私。
  - 不包含违法、暴力、色情或歧视性内容。
- 维护者保留对页面内容进行审核、修改或删除的权利，恕不另行通知。

## 使用方式

1. 直接访问仓库中的 `.html` 文件（可通过 GitHub Pages 或 Raw 链接查看）。
2. 克隆仓库到本地，在浏览器中打开任意 HTML 文件。
3. 如需引用页面，请注明来源仓库，但无需特别授权（除非页面内有额外声明）。

## 贡献指南

欢迎提交新的 HTML 页面，请遵循以下步骤：

1. Fork 本仓库。
2. 在合适位置添加您的页面文件。
3. 确保页面不含外部依赖（或已嵌入）。
4. 提交 Pull Request，简要说明页面用途。
5. 等待维护者审核合并。
